# こんだてLoop UI共通ルール（更新版）

## 目的
- 「本っぽい見た目」よりも「どの章/カテゴリを見ているかが直感で分かる体験」を優先する。
- 画面ごとの例外を増やさず、共通パーツ/ルールで統一感を保つ。

## トーン/基本方針
- **shadcn/ui前提**。セマンティックトークンのみ使用（直色指定禁止）。
- 余計な装飾は避け、**情報の主役がレシピ/セット**になる構成にする。
- **背景は基本白**。カテゴリは「線・タブ・ラベル」で示す。

## ヘッダー
- ヘッダーは**ヘッダーらしく見える**ことを優先（sticky + border + blur）。
- `HeaderBar` は `border-b` + `bg-background/90` + `backdrop-blur` で統一。
- 右側アクションは通知/冷蔵庫のみ（既存ルール通り）。

## ページタイトル
- **蛍光ペン風アンダーライン**を採用。
  - 線は水平、**端だけ斜め**（skew）で“ペン角度感”を出す。
  - 角丸は最小（`rounded-[1px]`）。
  - 色は**全ページ同色**（ページごとに色分けしない）。
  - **オレンジ系の薄色**を使い、主張しすぎない範囲でポップさを足す。
- サイズは `text-[22px]` + `tracking-tight`。

## レシピ帳/レシピカタログの“本”表現
- **本の面は画面下まで続く**（フッター下に隠れて終端が見えない）。
- 背景色の全面切替はしない。**白基調 + カテゴリ表現のみ**。

## レシピ/レシピセット タブ
- **左：レシピセット（規定） / 右：レシピ**。
- 見た目はフラット（押し込み線のみ許可）。
- **マイページ/通知のタブUIと同一パターン**で統一する。
- 下線などの区切り線は不要。

## カテゴリタブ（最重要）
- **カテゴリタブ + 縦線 + カテゴリ名ラベル**を「1つのデザイン」として扱う。
- 位置：左側・縦書き（`writing-mode: vertical-rl`）。
- タブ右側は**丸めない**（線と一体化させるため）。
- **タブ色と縦線色は同一**（透明感なし）。
- 選択中は**少し濃く**（brightness-95）＋軽いスライドで反応。
- **縦線は上端をタブ上端に揃える**。
- **縦線は下に伸ばし、フッターで隠れる**（末端が見えない）。
- カテゴリ名は本文左上に**小さな色付きバッジ**で表示（「カテゴリ:」表記なし）。

## カード（レシピ / レシピセット）
- サイズは **W: 140 / H: 230**（縦長固定）。
- 画像は**上半分**、情報は下半分。
- 角丸は控えめ（カードらしい形）。
- レシピセットは**縦長カードの束感**で表現（控えめな重なり）。
- **状態タグはカード横いっぱいにしない**（文字幅に合わせる）。
- ステータスバッジは**ボタンと見分けがつく**配色・サイズ。
- 献立表では**状態タグを出さない**。
- 作った状態は**グレーアウトではなく淡く**、**緑の「済」ラベル**で達成感を出す。

## ボタン/タグの区別
- 「保存/購入」は**ボタン**、状態系は**バッジ**として明確に区別。
- バッジは小さく、意味色を持つ（プロト準拠）。

## 検索・並び替え・絞り込み
- 検索は**丸型検索バー + アイコンボタン2つ**で統一。
- 並び替えは**「おすすめ順 / 新着順 / 人気順」**のモーダル。
- 絞り込みは**下から出るシート**で、**リセット/適用**を必ず持つ。
- 画面上部に**フィルタサマリーのチップ**を表示（並び替え含む）。

## 献立表のレシピ横並び
- スクロールバーは**非表示**。
- 左右に**矢印ボタン**を置き、ゲーム的に「左右へ送る」体験に寄せる。

## レシピ詳細 / セット詳細
- **レシピ詳細の順番**：料理名 → 作成者 → 出典元URL → 状態タグ → タグ → 何人前 → 材料 → 中間素材 → 作り方。
- **画面コンテキストでボタンが変わる**：
  - 献立表：`作った` を主ボタン。
  - レシピ帳：`ここからセットを作る / 外部共有 / 削除`。
  - レシピカタログ：`購入 / 保存 / ここからセットを作る`。
  - セット内から開いた場合は**「セット詳細に戻る」**を表示。
- **セット詳細**は概要/タグ/品数を出し、**中のレシピは横スクロールカード**で見せる。
- **購入直後の保存促しモーダル**は「保存する」**1ボタン**。閉じるのは**右上×のみ**。

## トースト
- トーストは**プロトの出方に統一**（下からふわっと浮き上がる）。
- クラスは `.toast-pop` を使用し、**fadeInUp系の動き**にする。

## セット作成 / レシピ追加
- セット作成は**既存セットから編集 → 名前/作成者/概要 → カバー → レシピ選択（1〜7品）**の順。
- レシピ選択は**モーダルでカード選択**し、最大7品で制限。
- レシピ追加は**URL/テキスト取り込み領域**を必ず持つ。

## 通知 / ニュース
- 通知の**ニュースは詳細画面**を持つ。
- あなた宛て通知はモーダル表示（簡易）でOK。

## 献立→買い物→冷蔵庫
- 買い物リストは**現在の献立セット**から生成。
- **冷蔵庫在庫の差分**のみを買い物リストに出す。
- 「作った」で**冷蔵庫在庫を消費**（足りない分は減らさない）。

## 冷蔵庫UI
- 一覧では**食材名のみ表示**（個数は表示しない）。
- **同じ名前の食材はグループ化**して1つにまとめる。
- 食材をタップすると**詳細モーダル**が開く。
  - 個数一覧を表示（例：「50g, 1個」）。
  - 各アイテムごとに**削除ボタン**を配置。
  - 削除時は確認ダイアログを表示。

## 色の整合性
- ページ内で使う色は必ず統一ルールで管理する。
- カテゴリタブ周辺は特に色が増えるので、**一貫性を最優先**する。
